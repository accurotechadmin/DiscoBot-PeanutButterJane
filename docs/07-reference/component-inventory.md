**Audience:** Maintainers, operators, and reviewers checking whether the implemented application pieces are present and connected.
**Status:** Current reference
**Last reviewed:** 2026-06-05
**Related files:** `../../README.md`, `../../composer.json`, `../../.env.example`, `../../bin/`, `../../config/`, `../../src/`, `../../tests/`, `../../storage/logs/.gitkeep`
**Related docs:** [Application at a glance](../00-start-here/application-at-a-glance.md), [Architecture overview](../03-technical-reference/architecture-overview.md), [File and directory reference](../03-technical-reference/file-and-directory-reference.md), [Class index](class-index.md), [Data flow index](data-flow-index.md)

This inventory maps each implemented component and sub-component to its source, tests, runtime relationships, and deeper documentation. It is intentionally limited to the current lightweight DiscordPHP CLI skeleton. It does not list frameworks, databases, queue workers, web controllers, retained log archives, external monitoring, or process-manager units because those are not implemented in this repository.
