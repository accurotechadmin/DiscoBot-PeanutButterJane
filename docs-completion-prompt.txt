### Startup and runtime

The bot starts from the command line with either:

```bash
php bin/bot.php
composer bot
```

The current runtime flow is:

1. `bin/bot.php` checks for Composer autoloading and exits with a clear STDERR message if dependencies are missing.
2. `bin/bot.php` loads `.env` values if a readable `.env` file is present.
3. `bin/bot.php` loads `config/bot.php` and `config/commands.php`.
4. `App\ConfigValidator::validateBotConfig()` validates the bot config before connecting to Discord.
5. `bin/bot.php` creates `App\ConsoleLogger`.
6. `bin/bot.php` creates `App\CommandRouter` from the command registry.
7. `bin/bot.php` creates `App\Bot`.
8. `App\Bot` creates the DiscordPHP client with default intents, adds `MESSAGE_CONTENT` for prefix/mention/DM paths, adds `DIRECT_MESSAGES` for DM commands, and uses `storeMessages => false`, `loadAllMembers => false`, and `retrieveBans => false`.
9. DiscordPHP owns the long-running event loop.
10. `App\Bot::run()` listens for `ready`, conditionally registers `Event::MESSAGE_CREATE` for enabled message-based paths, and registers/listens for slash commands when enabled.
11. `App\Bot` ignores bot messages and messages authored by the bot itself.
12. Message content is routed through `App\CommandRouter`.
13. Commands return string replies.
14. Non-empty replies are sent to the Discord message channel.
15. Reply-send failures are caught and logged as warnings.

### Command architecture

Preserve these current facts:

- `App\Commands\CommandInterface` defines `execute(CommandContext $context): string`, `description(): string`, and `usage(string $prefix): string`.
- Commands receive `App\CommandContext`.
- `CommandContext` contains nullable DiscordPHP `Discord` and `Message` objects so commands can be unit-tested without a live Discord connection.
- `CommandContext` also exposes `commandName()`, `arguments()`, `prefix()`, `config()`, `hasDiscord()`, and `hasMessage()`.
- `CommandRouter::route()` is the production DiscordPHP-message path.
- `CommandRouter::routeContent()` is the raw-content, offline-testable path.
- `CommandRouter` normalizes command names, handles aliases, injects command metadata into config, returns friendly unknown-command replies, and catches command exceptions with safe user-facing replies.
- `HelpCommand` reads injected `commands.metadata` from context config.
- Documentation examples under `docs/05-examples/` are not active commands unless the class is added under `src/Commands/` and registered in `config/commands.php`.

### Exact parser behavior to document accurately

Document and preserve the parser behavior implemented in `src/CommandRouter.php` and tested in `tests/CommandRouterTest.php`:

- Empty prefix ignores all content and returns `null`.
- Content is trimmed before prefix checks.
- Non-prefixed messages are ignored.
- Prefix-adjacent text is ignored, so `!botping` is not treated as `!bot ping`.
- Leading and trailing whitespace around a valid command is accepted.
- Multiple spaces and tabs between command and arguments are collapsed by whitespace splitting.
- A bare prefix routes to `help`.
- Command names are trimmed and lowercased.
- Arguments are split on whitespace into a list of strings.
- Aliases resolve before command lookup.
- Unknown commands return a friendly reply that preserves the normalized typed command name and suggests `<prefix> help`.
- Command exceptions are logged and return `Sorry, something went wrong while running that command.` without exposing implementation details to Discord.

### Logging facts

The logger writes console output and optional daily JSON files:

- `App\ConsoleLogger` writes timestamped log lines to STDOUT by default.
- It filters by `LOG_LEVEL` using `debug`, `info`, `warning`, and `error` severity order.
- Unknown log levels passed into the logger constructor are rejected.
- Unknown per-message log levels are treated as `info` by the logger.
- `storage/logs/.gitkeep` keeps the default generated daily JSON log directory present.
- Do not imply log aggregation, metrics, or monitoring exists.

Recommended link check script:

```bash
python3 - <<'PY'
from pathlib import Path
import re, sys
errors = []
for path in Path('docs').rglob('*.md'):
    text = path.read_text()
    for match in re.finditer(r'\[[^\]]+\]\(([^)]+)\)', text):
        target = match.group(1).split('#', 1)[0]
        if not target or target.startswith(('http://', 'https://', 'mailto:')):
            continue
        resolved = (path.parent / target).resolve()
        if not resolved.exists():
            errors.append(f'{path}:{text[:match.start()].count(chr(10)) + 1}: missing {target}')
if errors:
    print('\n'.join(errors))
    sys.exit(1)
print('All local Markdown links resolve.')
PY
```
